# GFlow Runner 插件开发检查清单

本文档提供了开发和验证 GFlow Runner 插件的完整检查清单。

## 1. 代码检查清单

### 1.1 Struct-based 插件

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 实现 `Task` 接口 | ☐ | 必须实现 Commit、Rollback、Destroy、String 方法 |
| `Commit` 方法签名正确 | ☐ | `Commit(ctx context.Context, request any) (any, error)` |
| `Rollback` 方法签名正确 | ☐ | `Rollback(ctx context.Context) error` |
| `Destroy` 方法签名正确 | ☐ | `Destroy(ctx context.Context) error` |
| `String` 方法返回任务名称 | ☐ | `String() string` |
| 实现 `TaskClone` 接口（可选） | ☐ | 用于有状态任务，支持实例克隆 |
| 添加依赖注入标签 | ☐ | 使用 `inject:""` 标签 |

### 1.2 Function-based 插件

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 函数签名符合规范 | ☐ | 参考支持的函数签名列表 |
| 正确处理 context 参数 | ☐ | context 作为第一个参数 |
| 正确处理错误返回 | ☐ | error 作为最后一个返回值 |
| 函数命名符合规范 | ☐ | 使用 PascalCase |

### 1.3 数据结构

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 请求结构体定义完整 | ☐ | 包含所有必要字段 |
| 响应结构体定义完整 | ☐ | 包含所有返回字段 |
| json 标签正确 | ☐ | 用于绑定 Properties |
| gflow 标签正确 | ☐ | 用于绑定 Headers 和 DataObjects |
| 字段类型正确 | ☐ | 支持基本类型、结构体、切片、Map |

| 检查项 | 状态 | 说明 |

## 7. 验收标准

### 7.1 功能验收

- [ ] 任务可以成功注册
- [ ] 任务可以正常执行
- [ ] 任务可以正确处理错误
- [ ] 任务可以正确回滚（如适用）
- [ ] 任务可以正确清理资源

### 7.2 性能验收

- [ ] 任务执行时间在预期范围内
- [ ] 任务内存使用在预期范围内
- [ ] 任务并发执行正常

### 7.3 安全验收

- [ ] 输入验证完整
- [ ] 无敏感信息泄露
- [ ] 错误处理不暴露内部细节

## 8. 常见问题检查

| 问题 | 检查项 | 解决方案 |
|------|--------|---------|
| 任务注册失败 | 检查任务名称是否重复 | 使用唯一的任务名称 |
| 依赖注入失败 | 检查依赖是否已提供 | 使用 `runner.Provide` 提供依赖 |
| 数据绑定失败 | 检查标签是否正确 | 确保 json 和 gflow 标签正确 |
| 执行超时 | 检查超时配置 | 增加超时时间或优化任务逻辑 |
| 内存泄漏 | 检查 Destroy 方法 | 确保 Destroy 正确释放资源 |
