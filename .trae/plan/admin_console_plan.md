# GFlow 后台管理页面功能规划

## 1. 项目概述

### 1.1 背景

GFlow 是一个基于 BPMN 2.0 规范的工作流引擎系统，目前提供了完整的 REST API 和 Swagger UI，但缺少一个友好的可视化后台管理界面。本规划旨在设计一个功能完善的后台管理系统。

### 1.2 现有系统能力

| 服务 | 功能描述 |
|------|----------|
| **BpmnRPC** | BPMN 流程定义管理、流程实例执行与监控 |
| **SystemRPC** | Runner 管理、端点管理、系统健康检查 |
| **AdminRPC** | 用户管理、角色管理、权限管理、路由管理 |
| **AuthRPC** | 用户认证、登录登出、个人信息管理 |

### 1.3 技术栈建议

- **前端框架**: React 18+ / Vue 3+
- **UI 组件库**: Ant Design / Element Plus
- **状态管理**: Redux Toolkit / Pinia
- **图表库**: ECharts / Recharts
- **流程设计器**: bpmn-js
- **HTTP 客户端**: Axios

## 模块一：仪表盘（Dashboard）

### 1. 系统概览

| 功能 | 描述 | API 接口 |
|------|------|----------|
| 系统状态卡片 | 显示系统运行状态、版本信息 | `GET /v1/ping` |
| 流程统计 | 运行中/成功/失败的流程数量 | `GET /v1/processes` |
| Runner 状态 | 在线/离线 Runner 数量 | `GET /v1/runners` |
| 今日执行量 | 今日流程执行次数统计 | `GET /v1/processes` |

### 2. 统计图表

| 图表 | 描述 | 数据来源 |
|------|------|----------|
| 流程执行趋势图 | 按时间统计流程执行数量 | Process 列表聚合 |
| 流程状态分布饼图 | 成功/失败/运行中占比 | Process 状态统计 |
| Runner 负载图 | 各 Runner 的任务执行量 | Runner 统计信息 |
| 端点调用热力图 | 各端点的调用频率 | Endpoint 调用统计 |

### 3. 实时监控

| 功能 | 描述 | 刷新频率 |
|------|------|----------|
| 流程执行实时日志 | 显示正在执行的流程 | 实时推送 |
| Runner 心跳状态 | Runner 在线状态监控 | 5秒刷新 |
| 系统资源监控 | CPU、内存使用情况 | 10秒刷新 |
| 错误告警列表 | 最近的执行错误 | 实时推送 |

## 模块二：流程管理

### 1. 流程定义管理

| 功能 | 描述 | API 接口 |
|------|------|----------|
| 流程定义列表 | 分页展示所有流程定义 | `GET /v1/definitions` |
| 流程定义详情 | 查看 BPMN XML 内容和元数据 | `GET /v1/definitions/{uid}` |
| 部署流程 | 上传 BPMN 文件部署新流程 | `POST /v1/definitions` |
| 删除流程 | 删除指定流程定义 | `DELETE /v1/definitions/{uid}` |
| 版本管理 | 查看流程的历史版本 | `GET /v1/definitions/{uid}?version=N` |

### 2. 流程实例管理

| 功能 | 描述 | API 接口 |
|------|------|----------|
| 流程实例列表 | 分页展示流程实例，支持筛选 | `GET /v1/processes` |
| 流程实例详情 | 查看执行状态、节点活动 | `GET /v1/processes/{id}` |
| 执行流程 | 手动触发流程执行 | `POST /v1/processes/execute` |
| 终止流程 | 强制终止运行中的流程 | 扩展接口 |
| 重试流程 | 重试失败的流程 | 扩展接口 |

### 3. 流程设计器

| 功能 | 描述 | 实现方式 |
|------|------|----------|
| BPMN 流程设计 | 可视化设计流程 | bpmn-js |
| 节点配置 | 配置任务节点属性 | 自定义属性面板 |
| 模板管理 | 保存常用流程模板 | 本地存储 |
| 导入导出 | 导入/导出 BPMN XML | 文件操作 |
| 流程预览 | 预览流程执行路径 | 流程模拟 |

### 4. 执行历史

| 功能 | 描述 | API 接口 |
|------|------|----------|
| 历史记录列表 | 查看历史执行记录 | `GET /v1/processes` |
| 执行详情 | 查看每次执行的详细信息 | `GET /v1/processes/{id}` |
| 节点执行轨迹 | 查看节点执行顺序和时间 | FlowNode 列表 |
| 输入输出数据 | 查看流程的输入输出 | Process Context |
| 错误日志 | 查看执行错误信息 | Process.err_msg |

## 6. 技术实现建议

### 6.1 前端项目结构

```
gflow-console/
├── public/
├── src/
│   ├── api/              # API 接口封装
│   ├── components/       # 通用组件
│   ├── layouts/          # 布局组件
│   ├── pages/            # 页面组件
│   │   ├── dashboard/    # 仪表盘
│   │   ├── process/      # 流程管理
│   │   ├── system/       # 系统管理
│   │   └── security/     # 安全管理
│   ├── stores/           # 状态管理
│   ├── hooks/            # 自定义 Hooks
│   ├── utils/            # 工具函数
│   └── types/            # 类型定义
├── package.json
└── vite.config.ts
```

### 6.2 关键技术点

1. **认证机制**: 使用 JWT Token，存储在 localStorage，请求时通过 Authorization Header 传递
2. **权限控制**: 基于路由和按钮级别的权限控制，根据用户权限动态渲染菜单和按钮
3. **流程设计器**: 集成 bpmn-js，自定义属性面板和工具栏
4. **实时更新**: 使用 WebSocket 或轮询实现实时数据更新
5. **错误处理**: 统一的错误处理机制，友好的错误提示
